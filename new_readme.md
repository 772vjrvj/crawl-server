[홈 서버 구축 배경 및 네트워크 세팅 요약]

1. 배경 및 문제 상황
- 이슈: 대상 사이트를 외부망(캐나다)에서 접속할 경우, 겉으로 보이는 웹 페이지 렌더링은 한국과 동일해 보임. 
   하지만 접속 국가(IP)에 따라 연결되는 백엔드 서버나 라우팅 환경이 다르게 구축되어 있어, 내부적으로 반환되는 API 응답 데이터가 한국 서버에서 호출할 때와 다름.
- 결과: 이로 인해 한국 내수용 데이터 구조에 맞춰 작성된 기존 크롤링 로직이 캐나다 환경에서는 유효한 데이터를 찾지 못하고 크롤링 실패로 이어짐.
- 대안 검토: 한국 IP 확보를 위해 '레지던셜 프록시(Resident Proxy)' 도입을 고려했으나 지속적인 유지 비용이 비싸서 보류.
- 최종 결정: 한국 내부에 남는 윈도우 노트북을 공유기에 연결하여, 한국 IP 기반의 크롤링 API 서버(Crawl-Server)를 직접 구축하기로 결정함.

2. 왜 KT 홈허브(공유기) 설정이 필수적인가?
- 외부 인터넷(캐나다)에서 내부 망(한국 집/사무실)으로 들어오려면 반드시 첫 번째 관문인 공유기(KT 홈허브)를 거쳐야 함.
- 기본적으로 공유기는 외부에서 들어오는 요청을 차단하거나 어느 PC로 보낼지 알 수 없음.
- 따라서 공유기에게 "외부에서 5000번 포트로 요청이 오면, 크롤링 서버가 있는 노트북으로 길을 안내해라"라는 규칙(포트포워딩)을 반드시 설정해야 함.

--------------------------------------------------

3. 현재 네트워크 상태
- 서버용 노트북 IP: 172.30.1.70 (★실제 서버 구동 타겟)
- 작업용 데스크탑 IP: 172.30.1.8 (관리용)
- 공유기(KT 홈허브) IP: 172.30.1.254

4. 발생 가능한 문제와 해결책
- 문제: 노트북 재부팅이나 와이파이 재연결 시 IP(172.30.1.70)가 유동적으로 변하면 포트포워딩 규칙이 어긋나 외부 접속이 끊어짐.
- 해결: 포트포워딩 설정 전, 노트북의 IP가 변경되지 않도록 '고정 IP'로 묶어두는 작업이 선행되어야 함.

5. KT 홈허브 포트포워딩 설정 값
- 외부 포트: 5000 ~ 5000
- 내부 IP 주소: 172.30.1.70 (★반드시 데스크탑이 아닌 노트북 IP 입력)
- 내부 포트: 5000 ~ 5000
- 프로토콜: TCP

🔍 공유기 관리자 페이지 주소 찾는 법
명령 프롬프트(CMD) 실행: 키보드에서 윈도우 키 + R을 누르고 cmd를 입력하거나, 시작 메뉴에서 cmd를 검색해 실행합니다.

명령어 입력: 까만 창에 ipconfig를 치고 엔터를 누릅니다.

'기본 게이트웨이' 확인: 결과 목록 중에서 기본 게이트웨이(Default Gateway) 옆에 적힌 숫자가 바로 공유기의 주소입니다.

예시: 기본 게이트웨이 . . . . . . : 172.30.1.254

💡 왜 '기본 게이트웨이'인가요?
네트워크에서 '게이트웨이(Gateway)'는 내 PC가 외부 인터넷 세상으로 나가기 위해 반드시 통과해야 하는 '현관문'을 뜻합니다. 가정집이나 소규모 사무실에서 이 현관문 역할은 100% 공유기가 담당하죠. 따라서 현관문의 주소가 곧 공유기 관리자 페이지로 들어가는 접속 주소가 됩니다.

🚨 만약 위의 기본 비밀번호로 로그인이 안 된다면?
과거에 보안을 위해 비밀번호를 직접 변경하셨거나, 특정 GIGA WiFi 모델일 수 있습니다. 이럴 때는 다음 두 가지 방법을 확인해 보세요.

방법 1: 공유기 스티커 확인
공유기 기기 본체(바닥면이나 뒷면)를 보시면 스티커가 붙어 있습니다. 거기에 '웹 관리자 비밀번호' 혹은 '초기 암호'라고 적힌 영문/숫자 조합이 있는지 확인해 보세요. (보통 공유기 와이파이 비밀번호와 관리자 페이지 비밀번호는 다릅니다!)

방법 2: 공유기 공장 초기화 (최후의 수단)
도저히 비밀번호가 기억나지 않는다면 공유기를 초기화해야 합니다.

공유기 뒷면에 있는 작은 [초기화] 또는 [Reset] 구멍을 찾습니다.

이쑤시개나 볼펜 끝으로 그 구멍을 약 5~10초간 꾹 누르고 있습니다.

공유기 앞면의 LED 불빛들이 전체적으로 깜빡이며 재부팅될 때 뗍니다.

2~3분 뒤 공유기가 완전히 켜지면, 다시 ktuser / homehub 로 로그인이 가능해집니다. (주의: 이렇게 하면 기존 와이파이 비밀번호 등도 초기화되니 주의하세요!)

로그인에 성공하셨다면 화면이 어떻게 보이는지 말씀해 주세요! 바로 포트포워딩 메뉴 위치를 짚어드리겠습니다.

내부 포트: 5000 ~ 5000

프로토콜: TCP

[참고: 공유기 IP가 2개인 이유]
* 172.30.1.254 (내부 게이트웨이): 집 안방(노트북)에서 거실(공유기)로 나갈 때 쓰는 '내부망 주소'
* 220.94.196.1 (외부 공인 IP): 공유기가 바깥세상(KT 인터넷)과 연결된 '외부 접속용 주소' (나중에 캐나다 클라이언트가 이 주소를 치고 들어옴)
